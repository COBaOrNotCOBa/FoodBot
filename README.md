# FoodBot

Консольное Kotlin-приложение (бот/сервис), в котором используется работа с сетью и локальной базой данных. Проект сделан как эксперимент: здесь пробуются HTTP-запросы, корутины и хранение данных в SQLite через SQLDelight.

## Что делает проект

- Выполняет HTTP-запросы к внешним API через `okhttp`
- Обрабатывает данные в формате JSON
- Хранит данные локально в SQLite при помощи **SQLDelight**
- Работает как обычное JVM-приложение с точкой входа `MainKt`

(Конкретную бизнес-логику можно при желании описать отдельным абзацем.)

## Технологический стек

- **Язык:** Kotlin (JVM), версия `2.0.20`
- **JVM:** JDK 17 (`jvmToolchain(17)`)
- **Сборка:** Gradle Kotlin DSL
- **Библиотеки:**
  - `kotlinx-serialization-json` — JSON-сериализация/десериализация
  - `okhttp` — HTTP-клиент
  - `kotlinx-coroutines-core` — корутины
  - `app.cash.sqldelight:sqlite-driver` — работа с SQLite через SQLDelight
- **SQLDelight:**
  - Конфиг: база `Database`
  - Package: `com.databases`

## Требования

- Установленный **JDK 17**
- Доступ в Интернет (если проект обращается к внешним API)
- **Gradle Wrapper** в проекте (обычно уже лежит в репозитории)

## Как запустить

### 1. Клонировать репозиторий

```bash
git clone https://github.com/COBaOrNotCOBa/FoodBot.git
cd FoodBot```
(Если репозиторий называется иначе — поменяй URL и каталог.)

2. Запуск через Gradle Wrapper
Linux / macOS:

bash
Копировать код
./gradlew run
Windows:

bash
Копировать код
gradlew.bat run
Приложение запустится с точкой входа MainKt, указанной в блоке application Gradle-скрипта.

3. Сборка дистрибутива (по желанию)
Можно собрать дистрибутив с готовыми скриптами запуска:

Linux / macOS:

bash
Копировать код
./gradlew installDist
Windows:

bash
Копировать код
gradlew.bat installDist
После этого появится каталог:

text
Копировать код
build/install/<имя-проекта>/
Внутри будет папка bin со скриптами запуска (<имя-проекта> / <имя-проекта>.bat) и папка lib с зависимостями.

Статус проекта
Учебно-экспериментальный проект: здесь обкатывается стек Kotlin + Coroutines + OkHttp + SQLDelight. Можно использовать как основу для дальнейших ботов/сервисов или для экспериментов с работой с БД и сетью.
